COMO O APP FUNCIONA:
--------------------
O app NÃO tem tela de login. Ele usa credenciais FIXAS:

1. Credenciais definidas em build.gradle.kts:
   - DEFAULT_PLAYER_API = "https://canais.is/player_api.php"
   - DEFAULT_USER = "max"
   - DEFAULT_PASS = "1h2yd90"

2. Na inicialização (MaxiApp.kt):
   - Carrega configurações salvas do DataStore
   - Se não tem configurações salvas, usa as credenciais padrão
   - Configura a API automaticamente
   - NÃO pede senha ao usuário

3. Fluxo de abertura:
   MaxiApp.onCreate()
   → SettingsRepo.loadBlocking()
   → XRepo.configure(api, user, pass)
   → MainActivity abre HomeScreen
   → HomeScreen carrega canais automaticamente

POSSÍVEL CAUSA DO CRASH:
-------------------------
O crash ANTES de abrir qualquer tela indica problema em:

1. MaxiApp.onCreate() - Inicialização do app
   - Pode estar travando no SettingsRepo.loadBlocking()
   - Pode estar travando no DataStore
   - Pode estar travando ao configurar Retrofit

2. MainActivity.onCreate() - Inicialização da UI
   - Pode estar faltando ícone/recurso
   - Pode ter erro no Compose

PARA ADICIONAR TELA DE LOGIN:
------------------------------
Seria necessário criar:
1. LoginScreen.kt com TextFields de username/password
2. Modificar HomeNav.kt para começar em "login"
3. Salvar credenciais após login bem-sucedido
4. Validar login com API antes de permitir acesso
